# Повний гайд по graph2Thread для захисту

---

## 1. ЩО ЦЕ ВЗАГАЛІ ТАКЕ?

**Предмет:** найімовірніше "Паралельне та розподілене програмування" або "Операційні системи".

**Суть завдання одним реченням:**
Програма дозволяє **намалювати блок-схему алгоритму** для кожного потоку (thread), а потім — автоматично **перетворити ці малюнки на реальний Python-код** із потоками, і **протестувати** його на всіх можливих варіантах виконання.

**Навіщо це потрібно в реальному житті:**
Багатопоточні програми — це коли кілька завдань виконуються одночасно (наприклад, браузер одночасно завантажує сторінку і грає відео). Але вони мають неприємну властивість: **результат залежить від того, хто першим встигне виконати свою дію** — це називається *race condition* (гонка потоків). Цей інструмент дозволяє **заздалегідь перевірити всі можливі сценарії** такої гонки.

## 4. ПОЯСНЕННЯ КОЖНОГО ФАЙЛУ

---

### `model.py` — Модель даних

Це **словник понять** проекту. Тут описано, що таке кожна сутність.

**Ієрархія:**
```
Project (проект)
  └── Flowchart × N  (блок-схема одного потоку)
        ├── Block × M  (один блок: прямокутник, ромб тощо)
        └── Edge × K   (стрілка між блоками)
```

**Типи блоків (`kind`):**

| kind | Вигляд | Що робить |
|---|---|---|
| `start` | овал зелений | початок потоку |
| `end` | овал червоний | кінець потоку |
| `assign` | прямокутник синій | `V = C` або `V1 = V2` |
| `input` | паралелограм жовтий | читає число зі stdin у змінну |
| `print` | паралелограм помаранчевий | виводить змінну в stdout |
| `cond` | ромб фіолетовий | розгалуження: `V == C` або `V < C` |

**Важливі константи:**
```python
INT_MAX = 2**31 - 1   # 2147483647 — максимальне значення змінної
MAX_THREADS = 100     # максимум потоків
MAX_BLOCKS = 100      # максимум блоків в одній блок-схемі
MAX_VARS = 100        # максимум змінних в одній блок-схемі
```

**Спільні змінні** — всі змінні з усіх блок-схем лежать у **спільній пам'яті** (один словник `MEM`), доступній кожному потоку. Ніяких "локальних" змінних немає.

---

### `interpreter.py` — Інтерпретатор

Це **серце логіки**. Описує, як виконується одна операція.

**Ключова ідея — стан програми (`State`):**
```python
State:
  mem        # { 'x': 0, 'y': 5, ... }  — значення всіх змінних
  pcs        # (3, 7, None)  — поточний блок кожного потоку (None = завершено)
  input_pos  # 2  — скільки чисел зі stdin вже прочитано
  output     # (0, 1, 5)  — що вже надрукували
```

**Стан незмінний** — кожен крок `step()` повертає **новий** стан, не змінюючи старий. Це дозволяє тестеру зберігати і відновлювати стани для перебору варіантів.

**Один крок (`step(state, tid)`):**
- бере поточний блок потоку `tid`
- виконує його (присвоєння / читання / друк / розгалуження)
- повертає новий стан із оновленим PC потоку

**Атомарність блоку** — поки потік виконує один блок, ніхто інший не може втрутитись. Потоки чергуються **тільки між блоками**.

### `translator.py` — Транслятор

Перетворює блок-схеми на **реальний Python-файл** із `threading`.

**Що генерується:**
```python
MEM = {'x': 0, 'y': 0}   # спільна пам'ять
LOCK = threading.Lock()   # один глобальний замок

def thread_0():           # один потік = одна функція
    pc = 1                # починаємо з START-блоку
    while pc is not None:
        with LOCK:        # захоплюємо замок на час одного блоку
            if pc == 1:   # START
                pc = 2
            elif pc == 2: # x = 5
                MEM['x'] = 5
                pc = 3
            elif pc == 3: # PRINT x
                _emit(MEM['x'])
                pc = 4
            elif pc == 4: # END
                pc = None
```

**Чому `with LOCK`?** — Щоб виконання одного блоку було атомарним. Це точно відповідає семантиці інтерпретатора (де один `step` теж атомарний).

### `tester.py` — Тестер

Найскладніша частина. Відповідає за **перебір всіх можливих порядків виконання потоків**.

**Чому потрібен перебір?**
Якщо є 2 потоки по 4 блоки — вони можуть чергуватись у **C(8,4) = 70 різних порядках**. Деякі дадуть `[0, 1]`, інші — `[1, 0]`. Тестер обходить усі 70.

**Алгоритм перебору — DFS по дереву розкладів:**
```
Корінь: початковий стан
  ├── крок потоку 0 → новий стан
  │     ├── крок потоку 0 → ...
  │     └── крок потоку 1 → ...
  └── крок потоку 1 → новий стан
        ├── ...
```
Кожен листок дерева — одне **повне виконання** (варіант).

**Класифікація листків:**
- `COMPLETE` — всі потоки завершились (дійшли до END)
- `DEADLOCK` — хтось застряг (наприклад, `INPUT` а вхід порожній)
- `OVER_BOUND` — перевищено ліміт кроків (нескінченний цикл)

**K-відсоток** (найскладніша частина завдання):
- `Den(K)` = скільки ВСЬОГО існує повних виконань із ≤K кроків (незалежний DFS з мемоізацією)
- `Num(K)` = скільки таких вже перебрано і перевірено
- `% = Num / Den × 100`

## 8. МОЖЛИВІ ПИТАННЯ НА ЗАХИСТІ

**"Що таке race condition?"**
> Це ситуація, коли результат програми залежить від того, який потік першим виконає свою операцію. Наприклад, два потоки одночасно читають і записують одну змінну — результат непередбачуваний. Саме такий приклад — `race_0_1.json`: A і B друкують різні числа, хто перший — невідомо.

**"Чому блок-схеми, а не просто код?"**
> Блок-схема — це візуальний спосіб задати алгоритм. Користувач не зобов'язаний знати Python, він просто малює логіку. Програма сама перетворює малюнок на код.

**"Що таке атомарність блоку?"**
> Один блок виконується повністю без переривань. Потоки можуть змінювати порядок виконання тільки між блоками, але не всередині одного блоку. У згенерованому коді це реалізовано через `with LOCK`.

**"Як забезпечується відсутність повторів при переборі?"**
> Ми використовуємо DFS-обхід дерева розкладів. Кожен шлях від кореня до листка — це унікальна послідовність виборів "який потік зараз ходить". Два різних шляхи по визначенню різні — повторів бути не може структурно.

**"Що таке K-відсоток?"**
> Коли ми перериваємо перебір, ми вже перевірили частину варіантів. K-відсоток показує: з усіх можливих коротких виконань (до K кроків) — скільки відсотків ми вже перевірили. Це дає кількісну оцінку покриття тестами на момент переривання.

**"Чому Python, а не C?"**
> Завдання дозволяло вибір мови. Python обрано тому що: 1) `tkinter` входить до стандартної бібліотеки; 2) `threading` модуль простий і зрозумілий; 3) код читається майже як псевдокод, що полегшує перевірку відповідності семантиці.

**"Що станеться якщо потік зациклиться?"**
> Тестер має параметр `op-bound` (ліміт кроків, за замовчуванням 1000). Якщо виконання перевищує ліміт — воно класифікується як `OVER_BOUND` і не вважається завершеним. Це захищає від нескінченного перебору.

**"Що таке deadlock у вашій програмі?"**
> Deadlock виникає коли жоден потік не може продовжити виконання. Найтиповіший випадок — блок `INPUT`, але вхідні дані вже закінчились. Такі виконання класифікуються як `DEADLOCK` і виключаються з підрахунку K-відсотка.
